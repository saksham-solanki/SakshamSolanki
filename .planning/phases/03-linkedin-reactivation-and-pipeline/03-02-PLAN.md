**posting-cadence.md — Define the complete daily posting system:**

Weekly schedule (7 days/week, 1 post/day):
- Assign each day a primary content pillar from the 70/30 mix
- 70% trending AI (5 days): Builder logs (Claude Code real usage), Explainer posts (MCP, AI agents for operators), Tool comparisons (data-driven AI tool tests), Vibe coding insights
- 30% authority (2 days): Case study results, Process insights, Contrarian takes
- Map format to content type: text-only for trending/tutorials, carousels (PDF) for case studies/frameworks/comparisons
- Include CTA strategy: which days get CTAs, which don't. DM-based only ("DM me 'audit'"). Not every post needs a CTA.
- Include engagement protocol: 15-20 min commenting before posting, reply to all comments within 15 min
- Include weekly batch production workflow: draft all 7 posts on Sunday, responsive to trending topics

Note: The cadence starts AFTER the 7-day warmup (Plan 03-01). This is the ongoing daily system beginning Day 8+.

**post-templates.md — Create reusable templates for each content type:**

Create 5 templates (one per content type):
1. **Builder Log template** — "I built X with [tool]. Here's what happened." Structure: hook → what I built → surprising finding → lesson → optional CTA
2. **Explainer template** — Break down trending topic for operators, not engineers. Structure: "Everyone's talking about X. Here's what it actually means for your business." → concept → practical application → takeaway
3. **Tool Comparison template** — Data-driven test of AI tools. Structure: "I tested X vs Y on a real project. Results:" → setup → metrics → winner → nuance
4. **Case Study Result template** — Business outcome from a real deployment. Structure: problem (with dollar cost) → what we built → result (with dollar impact) → lesson
5. **Contrarian Take template** — Challenge conventional AI wisdom. Structure: "Unpopular opinion: [contrarian claim]" → why everyone's wrong → what actually works → proof

Each template should include:
- Structure/outline (fill-in-the-blank sections)
- Example hooks (3 per template)
- Word count guidance (150-250 words)
- Format guidance (text vs carousel)
- CTA guidance (when to include, what type)
- What NOT to do (common mistakes for that content type)

Voice rules for all templates: operator-first, no banned words, no hashtags, specific over vague, metrics over adjectives.
